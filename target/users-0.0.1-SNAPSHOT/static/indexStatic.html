indexStatic.html sous ${context}/static

OBJECTIF : Vérifier que le Serveur Tomcat est bien démarré et que l'application a bien été déployée sous TOMCAT.

Cette page HTML ${context}/static/indexStatic.html située sous le répertoire static sous le contexte (webapp) ne sert qu'a vérifier le fait que les ressources hors WEB-INF sont accessibles pour les navigateurs (css, images, ...).

Cette page HTML "static/indexStatic.html" est accessible en tapant dans l'URL du navigateur soit :
  • http://host:8080:/${context} si le welcome-file est activé dans le /WEB-INF/web.xml
  • http://host:8080:/${context}/static/indexStatic.html si le welcome-file est désactivé dans le /WEB-INF/web.xml

[http://localhost:8080/contexte_appli_web/] n'est donc équivalent à [http://localhost:8080/contexte_appli_web/ static/indexStatic.html] que si le welcome-file vers static/indexStatic.html est activé dans le web.xml.

Le welcome-file exécute une redirection vers la première ressource valable (citée dans le <welcome-file-list> et présente sous le contexte) qu'il trouve.

Remarque : Les ressources statiques devant être servies en tapant leur URL directement dans le navigateur sont en réalité regroupées sous ${context}/static/ dans nos applications.

Utilisation de cette page ${context}/static/indexStatic.html

taper /static/indexStatic.html après le contexte dans l'URL du navigateur (par exemple http://localhost:8080/monApplication/static/indexStatic.html).

Implémentation des ressources de cette page ${context}/static/indexStatic.html avec chemin ABSOLU PAR RAPPORT au CONTEXT

Cette page utilise une css : ${context}/static/css/style_index.css.

La css est référencée en ABSOLU href="css/style_index.css". En absolu signifie par rapport au contexte.

L'absence de "/" au début de l'URL permet au serveur de résoudre le contexte : http://host:8080/context/
href="css/style_index.css" permet de décrire le chemin de la ressource css PAR RAPPORT A LA POSITION DE LA PAGE ACTUELLE à concaténer au contexte lors de la résolution par le serveur.
